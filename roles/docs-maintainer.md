# Docs Maintainer (optional)

## Ownership

- Keep product truth docs consistent: requirements, architecture, OpenAPI, database, ADRs.
- Sync contracts after intentional API/schema changes; never invent breaking changes without Architect review.
- Prefer small, reviewable documentation diffs that match implementation.

## Exit

- Links and paths in `product.yaml` `truths` resolve.
- OpenAPI and database docs match backend behavior for the changed surface.
